# Privacy Policy and Data Safety Checklist

## Privacy policy reference points
- Explain local-first storage of debts, payments, preferences, scenarios, and imported documents.
- Explain optional cloud extraction:
  - user-initiated only
  - OCR remains local-first
  - backend mediates provider access
- Explain encrypted full-backup export and replace-only restore.
- Explain protected preferences:
  - app lock
  - hidden balances
  - AI consent
  - relock timeout and screenshot/privacy controls
- Explain premium billing and backend entitlement verification.

## Google Play Data safety checklist
- Collected data:
  - user-provided financial/debt data
  - imported document text/images when the user imports them
  - purchase and entitlement metadata for billing
  - device/app integrity metadata for attestation
- Processing model:
  - local-first by default
  - cloud extraction only after explicit consent
  - no silent upload of statements/screenshots
- Security posture:
  - encrypted local database
  - encrypted document vault
  - passphrase-protected full backups
  - secure-storage protected preferences

## Items that still require external legal review
- final privacy policy URL
- jurisdiction-specific disclosure wording
- final retention/deletion wording for backups and imported documents
- final third-party service list for production telemetry and billing
